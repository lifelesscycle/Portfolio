The problem

Cafés don't run on spreadsheets.

TeenYaar Cafe needed one screen that could take an order, remember a regular's loyalty points, print a kitchen ticket, and still make sense to a staff member who's never touched a computer system before — without three different logins for three different jobs.

The build

One codebase, every role.

I built DineOS as a single React front end over an Express + MongoDB backend, packaged into an Electron desktop app so it feels like software the café owns — not a browser tab that can be closed by accident. Role-based access keeps a cashier's view lean while giving the owner the full picture.

What's inside

Six modules, one till.

Every screen below is wired to the same live database — a table opened in Quick Order shows up instantly in Kitchen View, and a bill settled at the counter updates Reports before the receipt finishes printing.

Floor

Table & order flow

A live floor plan — available, occupied, reserved, cleaning — feeding straight into dynamic table ordering with cart sync, so a table's running order never falls out of step with what's on the till.

Access

Role-based access control

Frontend filtering plus a backend requireRole middleware keep cashiers, kitchen staff and the owner each looking at exactly what their job needs — and nothing they could break.

Tabs

Credit ledger

Regulars run a tab. DineOS tracks what's outstanding, what's overdue, and what's been settled — turning "I'll pay next time" into a number the owner can actually see.

Stock

Inventory that deducts itself

Packaged products decrement automatically the moment they're sold through billing, so stock counts stay honest without anyone doing an evening recount.

Till

Billing, split by how they paid

Every bill lands in a searchable history, broken down by cash, UPI and card — the same three buckets the owner already thinks in.

Insight

Reports across any date range

Overview, profit & loss, item-wise, payment-wise, customer and credit reports — filterable from "today" to "last 30 days" and exportable to Excel or PDF for whoever does the books.

Under the hood

Built like a receipt — itemised, nothing hidden.

The stack behind the counter: a React front end talking to an Express API, printer hardware bridged over WebSockets, and the whole thing shipped as a real desktop install rather than a website pretending to be one.
